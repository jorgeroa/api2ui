# v1.2 Requirements

Requirements for Smart Parameters & Layout System milestone. Each maps to roadmap phases.

### URL Parsing

- [x] **PARSE-01**: Parse query params from any URL (not just Swagger/OpenAPI)
- [x] **PARSE-02**: Detect array params in both `tag[]=x` and `tag=x&tag=y` notations
- [x] **PARSE-03**: Group params by prefix (e.g., `ddcFilter[*]` â†’ "Filters" section)
- [x] **PARSE-04**: Infer basic types from values (string, number, boolean)
- [x] **PARSE-05**: Detect date/datetime formats in param values
- [x] **PARSE-06**: Detect email and URL formats in param values
- [x] **PARSE-07**: Detect coordinate pairs (latitude/longitude)
- [x] **PARSE-08**: Detect zip/postal codes

### Form Components

- [x] **FORM-01**: Date picker component for date/datetime fields
- [x] **FORM-02**: Multi-value tag input with chips for array params
- [x] **FORM-03**: Inline validation feedback (on blur, not keystroke)
- [x] **FORM-04**: Contextual placeholders with example values
- [x] **FORM-05**: Slider component for numeric ranges when min/max known
- [x] **FORM-06**: Grouped checkbox component for enum arrays

### Re-fetch Experience

- [x] **FETCH-01**: Smooth inline re-fetch without full page reload
- [x] **FETCH-02**: Loading states during fetch operations
- [x] **FETCH-03**: Error feedback with clear error messages
- [x] **FETCH-04**: Parameter persistence per-endpoint across sessions
- [x] **FETCH-05**: Applied filter chips showing active params
- [x] **FETCH-06**: "Clear all" button to reset params
- [x] **FETCH-07**: URL preview showing what will be fetched

### Layout System

- [x] **LAYOUT-01**: User-selectable layout presets
- [x] **LAYOUT-02**: Sidebar filters + main content layout
- [x] **LAYOUT-03**: Top filter bar + results below layout
- [x] **LAYOUT-04**: Split view layout (equal weight params/results)
- [x] **LAYOUT-05**: Collapsible drawer layout (filters on demand)
- [x] **LAYOUT-06**: Layout preference persisted per-endpoint
